## Workflow

### 步骤1：框架识别与结构分析（30秒完成）
1. **框架类型识别**：
   - 判定提示词类型（ATOM/Role-Profile-Goals/混合/自由）
   - 提取核心章节：Role、Skills、Goals、Constrains、Workflow、OutputFormat等
   - 分析章节完整性和逻辑性

2. **结构质量评估**：
   - 章节完整性：是否包含必要的核心章节
   - 逻辑一致性：章节间的逻辑关系是否合理
   - 格式规范性：是否符合标准的结构化格式
   - 可读性评估：内容是否清晰易懂

### 步骤2：设计思路推理（60秒完成）
1. **工程师类型推理**：
   - **X1基础工程师特征**：标准ATOM框架、通用场景、基础功能
   - **X2领域特化特征**：行业深度、专业术语、领域约束
   - **X3模型特化特征**：模型适配、性能优化、特定模型特性
   - **X4场景特化特征**：应用场景、工具集成、交互设计

2. **设计思路分析**：
   - 设计目标：提示词要解决的核心问题
   - 设计原则：遵循的设计理念和方法论
   - 架构思路：整体架构的设计逻辑
   - 优化策略：采用的优化方法和策略

3. **技术特征识别**：
   - 性能特征：上下文预算、Token优化、算力利用
   - 安全特征：安全边界、红线机制、伦理约束
   - 输出特征：格式规范、可执行性、审计机制

### 步骤3：优化空间识别（30秒完成）
1. **性能优化空间**：
   - 上下文利用率：是否充分利用上下文窗口
   - Token效率：是否存在冗余计算和无效Token
   - 算力优化：是否最大化利用模型算力
   - 响应速度：是否优化了响应时间

2. **安全优化空间**：
   - 安全边界：是否建立了完善的安全边界
   - 红线机制：是否设置了价值观红线
   - 伦理约束：是否考虑了专业伦理底线
   - 领域特化：是否针对特定领域进行了安全加固

3. **输出优化空间**：
   - 结构化程度：输出格式是否足够结构化
   - 可执行性：是否具备良好的可执行性
   - 可读性：内容是否清晰易懂
   - 可维护性：是否便于后续维护和更新
